# FocusFlow 产品需求文档（PRD）

> **版本**：v1.0 · **日期**：2026-06-22 · **状态**：Draft

## 1. 产品概述与目标用户

### 1.1 产品定义

FocusFlow 是一款 **本地优先的 Android 学习计划应用**，帮助用户制定计划、追踪进度、管理复习节奏。所有数据存储在本地，无需注册账号，安装即用。

核心解决的问题：自驱学习者"三天打鱼两天晒网"—— 计划模糊、执行无反馈、复习靠感觉、坚持没动力。


### 1.2 目标用户

| 用户类型 | 典型场景 | 核心痛点 |
|---------|---------|---------|
| **备考学生** | 考研/考公/语言考试，备考周期 3-12 个月 | 科目多不知道每天学什么，学了后面忘前面 |
| **自学转行者** | 利用下班/周末学编程、设计等新技能 | 时间碎片化，容易半途而废，缺乏阶段反馈 |
| **职场充电族** | 工作中需要持续学习新知识 | 学习时间有限需高效利用，缺少系统化管理 |

**共同特征**：有一定自驱力但缺乏外部监督，需要将模糊的学习意愿转化为可执行、可追踪的系统。

## 2. 功能列表

### 2.1 学习计划管理

| 功能 | 描述 |
|------|------|
| **创建计划** | 用户创建学习计划，设定名称、学科分类、起止日期 |
| **里程碑分解** | 每个计划可拆分为多个里程碑（阶段性目标），支持排序和截止日期 |
| **任务拆解** | 每个里程碑下创建具体任务，支持预估时长、优先级、截止日期 |
| **任务依赖** | 任务间可设置前置依赖，未完成前置任务时后续任务灰显 |
| **月计划** | 按月查看和规划：汇总本月目标任务、各里程碑预期进度、总学习时长目标 |
| **周计划** | 核心规划中枢：从任务池拖拽任务分配到本周各天，设定周目标，预览本周复习压力 |
| **日计划** | 每日执行清单：展示当天分配的任务 + 到期复习，支持临时增删调整 |
| **计划模板** | 提供预设学习路径模板（如"考研 180 天""GRE 90 天""Python 入门"），用户一键套用后可自由修改 |
| **计划状态管理** | 计划支持草稿/进行中/已完成/归档状态切换 |
| **学习记录（Session）** | 用户对任务启动计时器记录学习，支持暂停/继续，也可手动补录；可附加学习笔记 |

### 2.2 进度追踪与可视化

| 功能 | 描述 |
|------|------|
| **仪表盘** | 打开应用的默认页面，汇总展示 Streak、今日任务、计划进度、本周统计 |
| **学习热力图** | 参考 GitHub Contributions，每日学习时长映射为颜色深浅，一眼看出学习密度 |
| **计划进度树** | 树形展示 Plan → Milestone → Task 的完成百分比 |
| **数据统计** | 学习时长趋势图、学科分布饼图、任务完成率柱状图、效率分析 |

### 2.3 复习计划调度

| 功能 | 描述 |
|------|------|
| **复习计划** | 用户为任务设定复习周期（默认 1天/3天/7天/14天/30天，可自定义），系统自动计算下次复习日期 |
| **复习日历** | 日历视图展示每天需要复习的任务，提前预知复习压力 |
| **复习提醒** | 任务到达复习日期时推送提醒，仪表盘展示今日待复习数量 |
| **复习打卡** | 用户完成复习后标记"已复习"，系统自动安排下一轮复习；漏复习不重置，顺延至当天补复习 |
### 2.4 智能推荐

| 功能 | 描述 |
|------|------|
| **今日任务排序** | 综合截止日期紧迫度、优先级、依赖关系自动排序，推荐每天该先做什么 |
| **学习提醒** | 每日定时提醒学习、复习到期提醒（MVP 基础通知） |
| **智能提醒** | Streak 即将中断、里程碑进度落后、复习堆积时主动提醒（v0.2 规则引擎） |
| **下一步建议** | 基于当前进度和计划节奏，提示下一步应关注的内容 |

### 2.5 激励与习惯养成

| 功能 | 描述 |
|------|------|
| **连续学习（Streak）** | 每日学习 ≥ 5 分钟即打卡，连续天数实时展示；每月 2 次冻结机会防止一次中断归零 |
| **经验值（XP）** | 完成任务、学习时长、Streak 均获得 XP，按行为类型加权 |
| **等级系统** | 累计 XP 升级，纯展示激励，不影响功能 |
| **成就徽章** | 特定条件解锁（如"坚持 30 天""完成第一个计划"等） |
| **里程碑庆祝** | Streak 7/30/100/365 天时特殊动画提醒 |

### 2.6 学习报告

| 功能 | 描述 |
|------|------|
| **每日回顾** | 首次打开应用时展示昨日学习时长、完成任务数、Streak |
| **周报** | 每周自动生成，含时长对比、完成率、学科分布、薄弱点识别 |
| **月报 / 阶段报告** | 月度趋势分析、里程碑达成率 |
| **AI 摘要** | 基于学习数据生成自然语言总结和改进建议（需联网调用 LLM API，用户可选开启） |
| **报告导出** | 支持导出为 PDF / 图片 |

### 2.7 应用设置

| 功能 | 描述 |
|------|------|
| **学习偏好** | 每日学习目标（分钟）、复习提醒时间、Streak 冻结额度 |
| **数据管理** | 数据导出为 JSON 文件（用于备份/迁移）；从 JSON 文件导入恢复数据（导入时覆盖现有数据，需用户确认） |
| **通知设置** | 开启/关闭每日提醒、复习提醒、Streak 预警 |

## 4. 界面设计要求

### 4.1 设计原则

- **移动端优先**：主要使用场景是手机/平板，先保证小屏幕体验，再扩展到桌面
- **信息密度适中**：仪表盘一屏展示关键信息，不堆砌；详情页按需展开
- **即时正反馈**：完成任务、打卡、升级时有明确的视觉/动效反馈
- **克制的色彩**：主色调平静专注（推荐靛蓝/青色系），Streak/成就用暖色点缀

### 4.2 核心页面布局

## 5. 技术栈建议

### 5.1 整体架构

```
语言：    Kotlin
UI：      Jetpack Compose + Material Design 3
数据库：  Room（SQLite ORM，Jetpack 官方）
后台：    Foreground Service（计时器保活）+ WorkManager（定时任务/通知调度）
通知：    NotificationManager（原生 API）
导航：    Jetpack Navigation (Compose)
构建：    Gradle + Android Studio
最低版本：Android 8.0 (API 26)
```

### 5.2 选型理由

| 选择 | 理由 |
|------|------|
| **Kotlin** | Android 官方首选语言，空安全、协程、与 Java 完全互操作 |
| **Jetpack Compose** | 声明式 UI，开发效率高，Google 官方主推，动画/手势支持好 |
| **Room** | Jetpack 官方 SQLite ORM，编译期校验 SQL，类型安全，支持 Flow 响应式查询 |
| **Foreground Service** | 计时器在锁屏/切后台时持续运行，系统级保活，通知栏显示学习中状态 |
| **WorkManager** | 可靠的定时任务调度（复习提醒、每日统计），系统会智能合并省电 |
| **Jetpack Navigation** | Compose 原生导航，底部 Tab + 页面栈管理 |

### 5.3 项目结构（概要）

```
app/src/main/java/com/focusflow/
├── MainActivity.kt              # 入口 Activity
├── navigation/                  # 导航图 + 路由定义
├── ui/
│   ├── dashboard/               # 仪表盘页面
│   ├── plan/                    # 计划管理页面
│   ├── timer/                   # 计时器页面
│   ├── review/                  # 复习计划页面
│   └── settings/                # 设置页面
│   └── components/              # 共享 Compose 组件
├── data/
│   ├── db/                      # Room 数据库、DAO、Entity
│   └── repository/              # 数据仓库层
├── domain/                      # 业务逻辑（Streak 计算、复习调度等）
└── service/
    ├── TimerService.kt          # 前台服务（计时器保活）
    └── ReviewReminderWorker.kt  # WorkManager（复习提醒）
```

> **说明**：Gradle 构建产物为 APK/AAB，直接安装到 Android 设备。开发使用 Android Studio，运行在模拟器或真机。

## 6. 非功能性需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| 应用启动 | 冷启动 ≤ 2s，热启动 ≤ 1s |
| 页面切换 | Tab 切换 ≤ 300ms，无白屏 |
| 数据库查询 | 列表页加载 ≤ 200ms（千条级数据） |
| 热力图渲染 | ≤ 500ms（1 年数据量） |
| 计时器精度 | 误差 ±1 秒/小时 |

### 6.2 可用性

| 要求 | 说明 |
| 响应式 | 适配 Android 主流机型（360dp ~ 428dp 宽度），横竖屏均可用 |
| 离线即常态 | 数据完全存储在设备本地 SQLite，无需网络连接；在无信号环境（地铁/电梯）也能正常记录学习 |
| 无障碍 | 支持 TalkBack 屏幕阅读器，关键操作有内容描述 |
| 国际化 | 首版中文，预留 i18n 框架方便后续扩展英文 |

### 6.3 安全
| 要求 | 说明 |
|------|------|
| 数据隔离 | 所有数据存储在设备本地 SQLite，不经过任何服务器 |
| 最小权限 | 默认仅申请 `POST_NOTIFICATIONS`（通知）和 `FOREGROUND_SERVICE`（计时器保活）；网络权限仅在用户开启 AI 摘要功能时申请 |
| 输入校验 | 前端对所有用户输入做格式校验，防止异常数据写入本地数据库 |

### 6.4 数据

| 要求 | 说明 |
|------|------|
| 本地存储 | 所有数据通过 Room 持久化到设备本地 SQLite，无需联网，无服务端依赖 |
| 备份恢复 | 用户可随时导出全部数据为 JSON 文件，也可从 JSON 文件导入恢复 |
| 卸载保护 | 卸载应用会清除数据，提醒用户定期导出备份 |
| 隐私 | 数据完全在用户设备上，不采集、不上传、不共享 |
